SETUP FOR DUMMIES

This is the shortest path to a working local setup.
Everything can stay inside this folder.

What this setup does:
- downloads your IMAP mail into `./data/maildir`
- stores audit state in `./data/state`
- stores Kopia settings in `./data/kopia`
- generates the `mbsync` config automatically

What you need installed:
- `go`
- `mbsync`
- `kopia`

FIRST TIME SETUP

1. Run:

go run ./cmd/mail-backup setup

2. Answer the questions:

- IMAP host
- IMAP username
- IMAP password
- Kopia repository type
- local repo path or S3 settings

3. Test mail sync:

mbsync -c ./data/state/mbsyncrc.generated mail-backup

If that works, mail should appear in:

./data/maildir

4. Run the backup:

go run ./cmd/mail-backup backup

That will:
- sync mail again
- check for suspicious old-mail changes
- write a Kopia snapshot

USE IT AGAIN LATER

Normal usage is:

go run ./cmd/mail-backup backup

Or after building:

./mail-backup backup

FILES YOU CARE ABOUT

- `.env`
- `config.advanced`
- optional `config`
- `./data/maildir`
- `./data/state`

IMPORTANT

If you keep the Kopia repository on the same machine, this is not enough protection against total machine loss.
Prefer an off-host repository if the backup matters.
